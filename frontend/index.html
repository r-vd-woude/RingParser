RingParser

Map files to an XML XSD schema for Griel Bulkupload

Follow the steps below to upload your file, map it to the XSD schema, validate the mapping, and generate your XML file.

  • The file type will be automatically detected and column names will be used to suggest mappings. You can adjust the mappings as needed.
  • The delimiter for CSV files will be automatically detected.
  • For Excel files, the last active sheet is used by default.
  • Submit files are normal CSV files with the extension changed to .submit, and must include a column named euring2020.

If you have biometric data, you can include it in your file and map it to the biometric fields in the XSD schema. The biometric fields are optional, but if included, they will be validated and included in the generated XML.

Step 1: Upload File

Drag and drop your file here

Step 2: Select XSD Schema

Choose a built-in schema or upload your own .xsd file.

Step 3: Map Columns to Schema

Select the target fields from the XSD schema and map them to the columns in your uploaded file.

Fields in red with an * are required while white fields are optional.
You can also add biometric data if you have that.

On the right side of mapping interface, you will see a button with "✓ View sample".
Clicking that will show you what data is in the first few rows of the selected column.

Default fields

Biometric fields

Step 4: Validate Mapping

Validate your mapping against the selected XSD schema. This will check for required fields, data types, and any inconsistencies.

Validation results will appear here

Step 5: Generate XML

Once your mapping is validated, you can generate the XML file. You can preview the generated XML before downloading it.

If you uploaded a file with more than 1000 records, multiple XML files will be generated. Each file will contain up to 1000 records. You can download all generated XML files as a ZIP archive.
